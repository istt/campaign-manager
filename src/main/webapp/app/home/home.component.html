Features

Current workflow of Campaign Management Portal:

  • Broadcast Shortcode
  • Broadcast Contents
  • List of MSISDN: Can be retrieve from 3 sources:
    • Preload lists uploaded via CSV (re-useable)
    • One-time list uploaded directly during Campaign creation
    • 3rd list retrieve from other sources (API / Remote URL / Database...) lookup from IPGW.
    • Mutiple fields place holder support: Content provider may upload CSV field with first row is the header column, then use header as a placeholder for replace in every SMS send out. E.g, if a CSV have a column named msisdn, fullname, credit, then CP may write a message like ‘{{'Hello ${fullname}, your ${msisdn} credit is ${credit}.'}}’, then for each broadcast message, the correspondence value on each field will be replaced into placeholders.
  • CDR Information: All campaign metadata is recorded for cross checking
    • Service ID: Which VAS service this campaign target?
    • Content Provider ID: Which content Provider this campaign belongs to?
    • Service Provider ID: Which Service Provider this campaign belongs to?
    • Price: How much does it cost the CP / SP per one this particular message?
  • Workflow Management metadata:
    • CreatedBy, CreatedAt: Mark the username and timestamp which this campaign created.
    • ApprovedBy, ApprovedAt: Username and timestamp which this campaign approved.
    • Start DateTime, Expired Date Time: System should only select campaigns which are between run time, e.g StartTime < NOW() < Expired Time.
    • Running Hours: Only run campaign within these window timing. E,g 9,10,14,15 mean the campaign should run only at 9:00-11:00 AM and 14:00-16:00PM. Other hours it should be pause, to let the system idle during peak hours.
    • Running Weekdays: Campaign can be selected to run only on Monday - Saturday, but never on Sunday, for example.
    • Holiday: Forcefully stop all campaign in case current date is National Holiday, as it against the law. MVAS Operator Approve / Reject Pending Campaign At scheduled time, the campaign begin its running. During this stage, following options are available:
      • Pause / Resume: Let Operator pause the contents during peak hours, and Resume later on.
      • Terminate: Let operator to Terminate Paused Campaign, and mark it as Done.
    • During broadcasting, current MSISDN will be check against Do-not-disturb database. If the target number is in DND list, he will be skip.
    • After finish running, the system send report to configured email address to notify campaign results.
    • MVAS operator can view campaign info, download CSV result of Success Delivered messages, or Failed ones to retry (Retry = create new campaign with same content, not to re-upload the failed list on exists content)
    • Wholesales support features: Mobifone Admin can set the quota per Content Provider / Service ID / Service Provider, and use this as a categorize for wholesales biz model. E.g, sell 10000 broadcast messages for SP
  • Reporting Mechanism:
    • Per-campaign report: Number of message broadcasted, number of pending messages, number of response from customer in case of interactive campaign, statistic on percentages Customer Care Portal: In case customer complain about receive too many messages, the system is capable of doing customer care portal:
    • Look-up all messages send to one specified MSISDN, or MSISDN + date time range.
